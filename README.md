# huozhongche-skill

一个用于“AI火种车数智课堂”系列文章与推文写作的可复用 Codex Skill。

它不是一个单纯的“文案润色工具”，而是一套适合项目协作、系列内容生产和统一写作标准的写作系统。

## 仓库内容

- `SKILL.md`
  - Skill 主说明文件，定义完整写作流程与规则。
- `scripts/dedup_check.py`
  - 用于表达去重分析的辅助脚本。
- `references/`
  - 预留参考资料目录。
- `templates/`
  - 预留模板目录。
- `docs/skill-understanding.html`
  - Skill 结构理解说明页面。
- `docs/editor-training-deck.html`
  - 面向协作老师的系统讲解版页面。
- `docs/editor-training-deck.md`
  - 对应讲稿版说明。
- `docs/project-background.md`
  - 项目背景与命名更新说明。
- `examples/`
  - 预留案例目录，用于放精选写稿案例、示例稿与对照稿。

## 项目背景

`AI火种车` 目前已统一更新命名为：`AI火种车数智课堂`。

根据项目背景信息，这个公益项目：

- 由 WaytoAGI 教育版发起
- 旨在通过 AI 志愿老师将 AI 教育普及至偏远地区，缩小城乡数字鸿沟
- 项目首航于 2025 年 11 月在云南扎根，深入多民族聚居区开展“在地化”教学

因此，本仓库中的 Skill 可以理解为：围绕 `AI火种车数智课堂` 项目所沉淀的一套内容写作与协作规范。

更多背景说明可见：

- `docs/project-background.md`

## 这个 Skill 解决什么问题

它不是单纯教“怎么把活动写出来”，而是一套从素材梳理到终稿交付的写作系统，重点包括：

- 信息梳理与缺口识别
- 故事主线提炼
- 结构与标题风格选择
- 初稿深化与平视视角
- 配图策略与隐私保护
- 表达去重、错别字检查与终稿交付

更具体地说，它主要帮助解决以下几类问题：

- 素材很多，但不知道从哪里开始写
- 现场信息不完整，容易边写边猜
- 同一个系列写多了，文章越来越像
- 写完正文后，配图和结构还要重新返工
- 协作写稿时，大家对“什么是合格的 AI火种车数智课堂 风格”理解不一致

## 核心特点

- 把写稿过程拆成清晰的 6 个步骤，减少协作时的混乱感。
- 强调“平视视角”，避免怜悯叙事和扶贫标签化表达。
- 把配图建议纳入写作流程，而不是发稿前临时补图。
- 通过表达去重机制，避免系列稿件语言风格越来越像。

## 适用场景

这个 Skill 特别适合以下场景：

- 写“AI火种车数智课堂”项目相关的公众号文章、系列推文、阶段性报道
- 根据活动素材表、课表、采访记录、现场照片整理成正式稿件
- 多位老师或协作者共同参与写作，需要统一文章风格和检查标准
- 已经有一批发布过的稿件，希望新稿继续保持系列感，但又不重复
- 需要在写作过程中同步考虑配图选择、隐私保护和图文节奏

## 适合谁使用

- 项目内容编辑
- 协作写稿老师
- 系列内容统稿人
- 需要快速接手“AI火种车数智课堂”写作体系的新伙伴

## 不适合的场景

以下情况不太适合直接使用这个 Skill，或者需要先做额外调整：

- 完全不属于“AI火种车数智课堂”项目语境的通用营销文案
- 只需要一句短宣传语、海报标题或极短社媒 copy
- 没有素材表、没有活动信息、没有可核实背景的纯空写任务
- 不需要系列一致性、也不需要图文协同的普通随笔型写作

## 典型使用时机

你可以把它理解成一套“从接收素材到交付成稿”的写作流程。比较典型的使用时机包括：

- 活动刚结束，手头已有素材表和部分照片，准备开始写首稿
- 首稿写完，准备补标题方向、配图建议和文风统一
- 新稿准备上线前，需要和既往已发布稿件做表达去重
- 团队内部需要向新协作者解释“AI火种车数智课堂写作系统到底在要求什么”

## 推荐工作流

一个常见的使用方式是：

1. 先把素材表、活动课表、关键人物信息和照片整理齐
2. 用 Skill 做信息梳理与缺口识别
3. 确定主线、辅助线、叙事结构和标题方向
4. 完成初稿，并用平视视角、隐喻、细节等原则深化
5. 把配图建议直接嵌入稿件
6. 结合已发布稿件做表达去重，再进行错别字和一致性检查
7. 最后交付终稿与修改说明

## 如何使用

把本仓库作为本地 skill 安装到 Codex 技能目录后，在与“AI火种车数智课堂”相关的写作任务中调用即可。

如果你已经在使用本地 Codex skill 目录，通常结构类似：

```text
$CODEX_HOME/skills/huozhongche-skill
```

在实际调用时，更推荐把以下内容一并提供给模型：

- 素材表（如 `.xlsx`）
- 活动日期与课表
- 关键人物姓名与身份
- 已发布的相关文章或历史稿件
- 可用照片或图片说明

这些信息越完整，这个 Skill 的效果越稳定。

## 案例上传建议

如果后续要把写稿案例也上传进仓库，建议优先采用“精选案例”而不是一次性上传大体量原始包。

更推荐的方式是：

- 每次上传 1-3 篇代表性案例
- 优先上传可读版本，如 `.md`、`.pdf` 或体量适中的 `.docx`
- 如果需要保留原始大包，建议放云盘链接，而不是直接塞进 GitHub 仓库
- 案例文件可以按“场景 / 学校 / 阶段 / 类型”分类，方便后续检索和培训使用
