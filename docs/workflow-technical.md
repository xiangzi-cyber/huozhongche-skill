# 工作流 A：技术版（适合有 CLI 基础的用户）

> 适用人群：熟悉命令行操作、已配置飞书 CLI 环境的用户  
> 自动化程度：较高，表格读写通过 CLI 完成

---

## 流程总览

```
用户发素材 → 素材整理助手（AI） → 飞书 CLI 写入多维表格
                                          ↓
                                  飞书 CLI 读取多维表格
                                          ↓
                                  调用写文小助手（AI） → 推文初稿
                                          ↓
                                     审稿 → 发布
```

---

## 详细步骤

### 步骤 1：收集素材

老师在群聊里发的图片、文字、短视频，按环节保存到本地文件夹。

### 步骤 2：AI 整理素材

打开素材整理助手（任何支持看图的 AI 平台），按以下顺序操作：

1. 告诉它学校名称和活动日期
2. 上传课表
3. 按环节发送图片 + 文字说明
4. AI 输出「素材明细表」+「写稿亮点」

提示词见 `templates/material-agent-prompt.md`。

### 步骤 3：CLI 写入飞书多维表格

使用 `lark-cli base +record-upsert` 将 AI 整理好的素材写入飞书多维表格。

```bash
# 示例：写入一条素材记录
lark-cli base +record-upsert \
  --base-token <你的base_token> \
  --table-id <素材明细表的table_id> \
  --fields '{
    "活动编号": "HZC-2025-004",
    "学校名称": "XX小学",
    "环节": "教师AI讲座",
    "模块": "互动环节",
    "素材点": "老师积极举手",
    "具体内容": "教师AI小游戏中，老师们积极举手参与互动",
    "写稿价值": "展示活动形式的趣味性",
    "素材类型": "图片",
    "素材状态": "已整理"
  }'
```

批量写入时，可以将 AI 输出的表格内容转为 JSON，循环调用上述命令。

### 步骤 4：CLI 读取素材，调用写文助手

从飞书多维表格读取当前活动的素材：

```bash
# 读取指定活动的所有素材
lark-cli base +record-list \
  --base-token <你的base_token> \
  --table-id <素材明细表的table_id> \
  --filter '活动编号 = "HZC-2025-004"'
```

将读取结果作为输入，发给写文小助手（配合写作 Skill），按六步流程生成推文初稿。

写作 Skill 见 `SKILL.md`。

### 步骤 5：审稿与发布

1. 检查 AI 初稿，按需修改
2. 运行表达去重脚本（`scripts/dedup_check.py`）
3. 配图、排版、发布
4. 回飞书多维表格更新活动状态和发布链接
