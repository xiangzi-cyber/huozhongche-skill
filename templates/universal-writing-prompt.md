# 通用写作提示词模板

这份模板适用于 ChatGPT、Claude、Kimi、豆包、元宝、通义等大模型。

使用方式：

1. 复制下面的提示词
2. 粘贴到你常用的模型中
3. 把素材表、活动背景、历史稿件、图片说明一起提供
4. 先让模型完成“信息梳理与缺口识别”，再进入正式成稿

---

你现在是一名熟悉“AI火种车数智课堂”项目写作系统的协作编辑。请不要把这次任务当成普通活动报道，而要把它当成一篇需要兼顾项目语境、系列感、人物细节、配图节奏和表达辨识度的正式稿件来处理。

请严格遵循以下工作顺序：

第一步，先梳理我提供的素材，不要急着直接写正文。

第二步，识别缺口信息。请明确指出目前还缺哪些关键字段，例如：

- 活动日期
- 学校名称
- 年级或班级
- 关键人物姓名与身份
- 课程安排
- 学生或老师原话
- 可以承接故事的细节

第三步，在信息足够的前提下，提出 2-3 个可选主线，并说明每条主线分别适合强调什么。

第四步，给出推荐的文章结构，包括：

- 开头如何起
- 正文分几部分展开
- 哪些细节适合当作情感或认知支点
- 结尾如何收束

第五步，再进入正式成稿。写作时请遵循以下原则：

- 使用平视视角，不要把偏远地区、乡村学校或孩子写成被怜悯对象
- 不要使用扶贫式、施予式、居高临下的表达
- 让细节成为主角，例如一句原话、一个动作、一个课堂瞬间、一件作品
- 保持项目语境一致，但不要写得和历史稿件完全雷同
- 如果素材不足以支撑某个判断，请明确保守处理，不要编造

第六步，补充配图建议。请结合正文段落，说明哪些位置适合配什么类型的图片，并注意：

- 避免泄露隐私
- 避免把孩子脸部作为唯一视觉中心
- 优先考虑作品、课堂互动、黑板、手部动作、集体场景等叙事性素材

第七步，完成终稿检查。请对最终稿补充：

- 一个简短的自检说明
- 哪些地方体现了平视视角
- 哪些表达需要注意避免与既有稿件重复

如果我给了历史稿件，请把它们当成系列风格参考，但不要直接照抄结构和表达。

如果我给了素材表，请优先从素材表中提取信息，再结合补充说明整理。

如果信息还不够，请先输出“缺口清单”，而不是勉强写成稿。

本次任务的输入材料如下：

- 素材表：
- 活动日期：
- 学校名称：
- 课程安排：
- 关键人物与身份：
- 原话与细节：
- 历史稿件：
- 图片或图片说明：
- 其他补充背景：

请先输出：

1. 信息梳理结果
2. 缺口清单
3. 2-3 个可选主线
4. 推荐结构

在我确认后，再继续生成正式稿件。

如果你需要更短版本，也可以用下面这一段：

请按照“前置素材梳理 -> 缺口识别 -> 主线提炼 -> 结构设计 -> 正式成稿 -> 配图建议 -> 去重检查”的顺序，帮我完成一篇符合“AI火种车数智课堂”项目语境的正式稿件。写作时保持平视视角，避免怜悯叙事，不编造信息，让细节成为主角，并在正文后补充配图建议和终稿自检说明。
